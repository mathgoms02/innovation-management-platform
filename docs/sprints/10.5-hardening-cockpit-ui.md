# Sprint 10.5: Hardening, Cockpit do Organizador e Refino de UI

## Objetivo
Sprint intermediária (antes da Sprint 11) para fechar lacunas críticas descobertas em auditoria do projeto: corrigir falhas de segurança/consistência, destravar o papel de Organizador/Admin no frontend (que só existia no backend) e elevar a experiência de navegação e o acabamento visual cyberpunk.

## Pilares

### A — Hardening de Segurança & Correções [CONCLUÍDO]
- [x] **#1 Escalonamento de privilégio (crítico):** `RegisterSerializer` removeu o campo `role`; auto-cadastro é sempre `PARTICIPANT`. A tela de registro não oferece mais "Jurado". Coberto por teste (`test_register_ignores_privileged_role`).
- [x] **#2 Refresh de token + 401:** interceptor de resposta em `services/api.ts` que renova o access via `/users/token/refresh/`, enfileira requests concorrentes durante a renovação e, ao falhar, limpa a sessão e redireciona ao login.
- [x] **#3 Ownership de Announcement/Criterion:** `Announcement` ganhou `created_by` (setado no `perform_create`); `IsAdminOrOrganizerOrReadOnly._owns` valida dono por `organizer`, `created_by` ou `hackathon.organizer`.
- [x] **#4 Submissão:** `SubmissionViewSet.create` valida via serializer (`raise_exception`) e captura apenas `ValidationError`; removido o `UniqueValidator` implícito do `team` (conflitava com update-or-create).
- [x] **#5 Avaliação:** escala 0–10 e remoção dos comentários de desenvolvimento.
- [x] **#9 Equipes:** endpoints `leave` (membro sai; líder não pode) e `members/<id>/remove` (líder remove); desfazer = DELETE restrito ao líder. Cobertos por testes.

### B — Cockpit do Organizador (UI) [CONCLUÍDO]
- [x] Página `/manage` (ADMIN/ORGANIZER) com CSS Grid: criar/editar hackathon, gerenciar critérios + pesos, designar jurados (multi-seleção) e publicar anúncios globais. Organizador vê apenas os próprios eventos; ADMIN vê todos.
- [x] Endpoint `GET /api/users/?role=` restrito a Admin/Organizador para listar jurados. `CriterionSerializer` passou a expor `hackathon` (criação de critério via API estava quebrada).

### C — AppLayout + Refactor de Navegação [CONCLUÍDO]
- [x] Componente `AppLayout` com sidebar role-aware (item "Gerenciar" só para ADMIN/ORGANIZER) e topbar persistente (busca/autocomplete de hackathons, notificações, menu de usuário).
- [x] Todas as páginas migradas para o `AppLayout`, removendo headers/"voltar" duplicados.

### D — Polimento Visual Cyberpunk [CONCLUÍDO]
- [x] Fundo `cyber-grid` (blueprint sutil + vignette/glow radial), glow no item de nav ativo, borda com gradiente animado (`gradient-border`), scrollbar temática e animações (`animation-fade-in`/`slide-up`) — que eram referenciadas mas não existiam.

## Diretrizes
- Service Layer e validação dupla (API + frontend) mantidos.
- CSS Grid nos layouts complexos; `import type`; tema cyberpunk (`#0b0c10`, cyan `#00f0ff`, magenta `#ff007a`).
- Sem novos erros de lint/build; testes de backend verdes.
